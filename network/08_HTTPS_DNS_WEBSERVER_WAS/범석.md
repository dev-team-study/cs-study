# **통신**

## HTTPS, SSL/TLS

### HTTPS 

> HyperText Transfer Protocol Secure의 약자로, HTTP에서 보안이 강화된 프로토콜입니다.
>
> 기존의 HTTP는 통신하는 당사자 외에도 모든 사람들이 알아 볼 수 있는 반면, HTTPS는 통신하는 당사자들만 알아볼 수 있도록 암호화 해서 통신을 하는 방식입니다. 

### SSL / TLS

> SSL : Secure Sockets Layer // TLS : Transport Layer Security
>
> TLS는 SSL 3.0 이후 버전들을 말한다. 
>
> SSL은 클라이언트와 서버가 서로 데이터를 암호화 해 통신할 수 있도록 돕는 보안 계층이다. 
>
> OSI 각 계층은 독립화 되어있기 때문에, SSL 또한 HTTPS뿐만 아니라 다른 프로토콜과 조합해서도 사용할 수 있다. 

### 암호화 방식

> 대칭키 
>
> > 하나의 키로 암호화 복호화를 둘 다 할 수 있는 암호화 기법. 
> >
> > 공개키 기법보다는 빠르지만, 키를 안전하게 교환하기 어렵다는 단점이 존재한다. 
>
> 공개키 
>
> > 서로 다른 키로 암호화, 복호화를 하는 기법.
> >
> > 대칭키에 비해서는 안전하지만, 암호화 과정이 복잡해 속도가 비교적 느리다. 

## DNS

> Domain Name System의 약자로, 
>
> 숫자로 된 인터넷 프로토콜 주소 대신 인터넷 도메인 이름과 검색 가능한 URL을 사용해 웹사이트에 접속 할 수 있게 해주는 시스템. 기본적으로 DNS는 인터넷 상의 주소록이며, 사람은 도메인의 이름을 통해 웹사이트나 온라인 서비스에 액세스할 수 있다. 

## 기타 : socket, STOMP, SMTP (프로젝트에서 사용한 경우)

### Socket 

> 양방향 채널을 이용해 채팅방 처럼 양방향 통신이 가능하다. 
>
> 최초 접속이 일반 http request를 통해 handshaking 과정을 통해 이루어지기 때문에 기존의 80, 443 포트로 접속을 하므로 추가로 방화벽을 열지 않고도 양방향 통신이 가능하고, http규격인 CORS적용이나 인증등의 과정을 기존과 동이랗게 가져갈 수 있다. 

### STOMP

> STOMP는 Simple Text Oriented Messaging Protocol의 약자이다. 
>
> 간단히 메세지를 전송하기 위한 프로토콜로 메시지 브로커와 publisher-subscriber 방식을 사용한다. 
>
> 메세지의 발행자와 구독자가 존재하고, 메세지를 보내는 사람과 받는 사람이 구분되어 있다. 
>
> 메세지 브로커는 발행자가 보낸 메시지를 구독자에게 전달해주는 역할을 한다. 

# **Web**

## Web Server vs WAS

### Web Server

> 웹 서버란 사용자가 웹 브라우저에서 어떠한 페이지 요청을 하면 웹 서버에서 그 요청을 받아 정적 컨텐츠를 제공하는 서버이다.
>
> 여기서 정적 컨텐츠란 HTML, CSS, Javascript, 이미지, 파일 등 즉시 응답 가능한 컨텐츠이다.
>
> 만약 웹 서버가 동적응답을 받는다면, WAS에게 해당 요청을 넘겨주고, WAS에서 처리한 결과를 클라이언트에게 전달해준다.

### Web Application Server

> 웹 서버와 웹 컨테이너가 합쳐진 형태로, 웹 서버 단독으로는 처리할 수 없는 데이터베이스 조회나 다양한 로직 처리가 필요한 동적 컨텐츠를 제공한다. WAS는 JSP, Servlet 구동 환경을 제공해주기 때문에 웹 컨테이너 혹은 서블릿 컨테이너라고도 부른다.
